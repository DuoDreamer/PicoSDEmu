# Contributing to PicoSDEmu

## Scope and target

PicoSDEmu targets the Raspberry Pi Pico 2 / RP2350. The initial product is an
SD card emulator in SPI mode; native SD modes and non-Pico-2 boards are outside
the current scope unless the feature specification is deliberately updated.

## Original-work and licensing policy

Every contribution must be original work created for PicoSDEmu and compatible
with the repository's MIT license.

- Do not copy or adapt source code, tests, scripts, PIO programs, diagrams, or
  documentation from another repository, tutorial, SDK example, or product.
- Do not add vendored source, binary blobs, generated files, package archives,
  or dependencies without explicit prior approval and an entry in
  `docs/references.md`.
- Use published standards and hardware datasheets to understand required
  behavior, but write PicoSDEmu's implementation, test data, and explanatory
  text independently.
- Keep committed test media small and project-owned. Prefer deterministic data
  generated by a checked-in project script over binary disk images.
- Record any newly relied-on external document, tool, SDK, library, or runtime
  dependency in `docs/references.md` before it becomes required.

By submitting a contribution, the author states that they have the right to
contribute it under the MIT license and that it complies with this policy.

## Implementation expectations

- Use C++17 and the project's warning settings. Keep transport, SD-model, and
  host-file logic testable without Pico SDK headers where practical.
- Do not allocate dynamically, block, or emit debugging output from PIO, DMA,
  interrupt, or client-SPI timing-critical paths.
- Preserve the safe-absent behavior: when no valid backend is mounted, do not
  intentionally drive the client SPI bus.
- Add tests with every behavior change. Tests must demonstrate the intended
  behavior rather than merely duplicate the implementation.
- Do not weaken, delete, or skip a failing test to make a phase gate pass.

## Before submitting

Run the applicable native build and test commands:

```sh
cmake -S . -B build/host -DBUILD_TESTING=ON
cmake --build build/host
ctest --test-dir build/host --output-on-failure
git diff --check
```

Firmware changes additionally require a Pico SDK configuration and the
hardware/logic-analyzer checks specified by the current implementation phase.
